pipa.sh lets you expose an SSH server from behind NAT with a reverse SSH connection, then reach it through standard OpenSSH ProxyJump.

You do not have to memorize the flow. pipa.sh prints the next step when you register a host, when you publish it, and when a new client lands on the service.

View the repo See quick start

Quick Start

Three steps

The relay has one job: connect a public SSH client to a live SSH server running somewhere else.

1

Register a hostname

Connect to pipa.sh, get a random relay hostname, and let the service print the publish command for you.

ssh pipa.sh
2

Publish from the machine behind NAT

Run the printed reverse SSH command on the machine that has the SSH server you want to expose.

ssh -R 22:localhost:22 <token>@pipa.sh
3

Connect through ProxyJump

Point OpenSSH at the relay once, then connect to published hosts normally. New clients also get setup directions if they hit the service directly.

Host *.pipa.sh
    HostName %h
    ProxyJump pipa.sh

Publisher flow

$ ssh pipa.sh
...
Your relay hostname is ready:

  x7k2m4q9pa.pipa.sh

Run this on the machine that has the SSH server you want to publish:

  $ ssh -R 22:localhost:22 fz6rvtz2w6aj76my5gjzqqum@pipa.sh

What it is

pipa.sh is a small SSH relay for intentionally public hosts. It keeps the routing logic in the relay and leaves final authentication to the published SSH server.

What it is not

It is not a shell host, not a web control plane, and not a generic port-forwarding service to the wider Internet.

Notes

Basic behavior

Custom domains

You can point your own SSH hostname at a published *.pipa.sh host with a CNAME. The relay follows the CNAME chain and routes only if the final name is a registered, live publisher.

Bearer token model

The generated publish command contains a token. Possession of that token is enough to publish the hostname, so treat the full command like a secret.

Source

The implementation is intentionally small and readable. For the server, test harness, and deployment notes, use the GitHub repo.

github.com/alecdotninja/pipa
